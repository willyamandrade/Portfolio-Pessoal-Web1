Projeto - Portfólio Pessoal

1 Objetivo do Projeto
Criar uma página de portfólio pessoal utilizando HTML e CSS, aplicando corretamente os conceitos

estudados ao longo do módulo, incluindo semântica, formulários, tabelas, listas, imagens, posiciona-
mento e Flexbox.

A atividade é individual e cada aluno deverá publicar o site no GitHub Pages e enviar o link para
avaliação via Google Classroom.
2 Requisitos Técnicos
2.1 Estrutura da página principal (index.html)
1. Estrutura semântica
• <header> com o nome completo e navegação principal (<nav>).
• <main> com as seções principais do site.
• <footer> com informações de contato (e-mail, GitHub, etc).
2. Navegação
• Links internos no <nav> que levem às seções da página (exemplo: “Sobre mim”, “Projetos”,
“Contato”).
3. Seção "Sobre mim"
• Um pequeno texto de apresentação (usando <p> e h1, h2, ...).
• Uma foto pessoal (usando <img> com <figure> e <figcaption>).
4. Seção "Projetos"
• Lista (<ul> ou <ol>) com nomes de projetos ou trabalhos feitos em disciplinas (que você
ache relevante) até agora.
• Cada item pode ter um link (<a>) que leva a outro site (ex: repositório no GitHub com
projeto de POO).
5. Seção "Habilidades"
• Uma tabela (<table>) listando as linguagens ou tecnologias que você conhece.
Exemplo: Colunas “Tecnologia”, “Nível de Conhecimento”, “Tempo de Experiência”.
6. Seção "Contato"
• Um formulário (<form>) com os seguintes campos obrigatórios:
Nome (input type="text")
E-mail (input type="email")
Assunto (input type="text")
Mensagem (textarea)
Botão de envio (button ou input type="submit")

2.2 Requisitos de Estilo (CSS)
• O arquivo style.css deve incluir:
– Uso de seletores por tag, classe e ID.
– Definição de cores e tipografia (fontes, tamanhos, pesos, etc.).
– Aplicação correta do box model (margem, padding, borda).
– Uso de posicionamento para organizar elementos (ex: relative, absolute, fixed ou
sticky).
– Uso de Flexbox para organizar seções e/ou cards de forma responsiva (exemplo: área de
projetos ou habilidades).
– Um layout agradável, com harmonia entre as cores e boa legibilidade.
– Uso do seletor :hover em pelo menos um elemento (ex: botões ou links).

2.4 Dicas
• Use cores e fontes que combinem com sua personalidade.
• Adicione ícones (ex: Font Awesome).
• Teste a página em diferentes tamanhos de tela (computador, celular, tablet, etc).
• Capriche na ortografia e na organização do código.
